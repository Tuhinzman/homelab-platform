## Issues encountered + resolutions

### Issue 1 — Same-terminal "drill" (early morning)
- **Symptom:** Initial drill ran in same terminal, OS cache may have contaminated test
- **Resolution:** Mini cold-mount gate added explicitly — Cmd+Q Terminal, fresh launch, then test
- **Permanent fix:** Documented as Rule 3 in runbook

### Issue 2 — Permission anomaly on sparsebundle
- **Symptom:** `cp -Rp` to sparsebundle reset all files to `rw-------` (sparsebundle's noowners flag)
- **Resolution:** Explicit chmod after copy (644 for public certs, 600 for keys)
- **Documented:** Step 8 in runbook

### Issue 3 — `security verify-cert` confusing output (CSSMERR_TP_CERT_SUSPENDED)
- **Symptom:** Mac native verify-cert flagged cert as "suspended" with CT log warnings
- **Cause:** macOS framework expects public CA standards (CRL, CT logs); internal CA has neither
- **Resolution:** `openssl verify` with explicit CAfile = real chain validation; verify-cert framework irrelevant for internal CA use cases
- **Documented:** Rule 4 hash format clarity, Phase I notes

### Issue 4 — bash heredoc append vs overwrite
- **Symptom:** `cat > file <<EOF` appended to existing content instead of overwriting
- **Cause:** First-line `cat > ...` not properly captured during paste
- **Resolution:** Re-run with extra paste discipline; verified clean overwrite via head/tail
- **Lesson:** File-write commands always include verification chunk after

### Issue 5 — zsh syntax incompat (`read -p`)
- **Symptom:** `read: -p: no coprocess` error during Keychain entry creation
- **Cause:** Bash `read -p prompt VAR` syntax invalid in zsh; zsh uses `read "VAR?prompt"`
- **Resolution:** Rewrote using zsh-native syntax, length-check sanity gate added
- **Lesson:** Always verify shell flavor before assuming syntax

### Issue 6 — Keychain scope deviation (security model)
- **Symptom:** Created 3 Keychain entries including ROOT-KEY + SIGNING-KEY; presented as "convenience"
- **Pushback:** Live online encrypted private key passphrases = compromised root if Mac compromised
- **Resolution:** Deleted ROOT-KEY + SIGNING-KEY entries, retained only SPARSEBUNDLE
- **Permanent rule:** Documented as Rule 2 in runbook
- **Reflection:** Caught late-session by Tuhin; Claude had drifted on threat model

2. Backup -> cold-mount drill (Mini Gate, then full HARD GATE C)
